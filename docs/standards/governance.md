# 强约束治理规范

## 必须遵守

1. 必须优先选用标准组件/标准页面。
2. 不允许手写列表页、表单页、详情页骨架。
3. 不允许绕过统一 theme / i18n / alerts / permission / org-context。
4. 标准组件库的 UI / 主题 / 样式 / 响应式基线统一遵循 **Material Design 3（MD3）**，当前实现基座统一按 **Vuetify 体系** 收敛。
5. 新页面必须声明 page type。
6. 新组件必须说明标准库为何无法满足。
7. 例外必须受控、可审计、可过期回收。

## CI 最低要求

- 校验页面清单中的 `pageType`
- 校验必须组件是否齐全
- 校验禁止绕过的基础能力
- 校验例外申请是否完整
- 校验标准组件/页面 registry 没有被随意删除
